Overview

GD Banner Automator is a Figma plugin designed to streamline the creation of multiple banner ad variants within Figma. It automates the process of generating similar designs based on structured data from spreadsheets, significantly reducing manual effort and ensuring consistency across banner sets.

The plugin allows you to:

  • Upload data from Excel/CSV files
  • Map spreadsheet columns to Figma layers
  • Generate multiple banner variants automatically
  • Export the generated banners as image files
GD Banner Automator interface

Installation

Install from Manifest File

The GD Banner Automator plugin is installed using a manifest file:

  1. Download the plugin files from the Gera Dovana SharePoint: Download Plugin Files
  2. In Figma, click on the menu icon (≡) in the top-left corner
  3. Go to "Plugins" > "Development" > "Import plugin from manifest..."
  4. Navigate to and select the manifest.json file from the downloaded plugin files
  5. Click "Open" to complete the installation
Note

When installed from manifest, the plugin will appear in the "Development" section of your plugins menu. To access it, navigate to Plugins > Development > GD Banner Automator.

Once installed, you can access the plugin from the Plugins > Development menu in Figma.

Preparing Your Figma Document

Before using the plugin, you need to set up your Figma document properly:

  1. Create or open a document containing templates: Design your banner templates on a dedicated page in your Figma document. These will serve as the base templates for generating variants.
  2. Naming convention: Use clear, descriptive names for your layers that correspond to the data fields in your spreadsheet. For example, if you have a column named "Product_Name" in your spreadsheet, name the corresponding text layer "Product_Name" in your Figma design.
  3. Special naming for price layers: For advanced price formatting, use the following naming convention:
    • price_[ColumnName]_main - For the main part of the price (e.g., "12")
    • price_[ColumnName]_decimal - For the decimal part (e.g., "99")
    • price_[ColumnName]_currency - For the currency symbol (e.g., "€")
  4. Image layers: For layers that will display images from URLs in your spreadsheet, ensure they can accept image fills (rectangles, ellipses, frames, etc.).
  5. Visibility layers: Layers that will be shown/hidden based on data should have names that match the corresponding column in your spreadsheet.

Preparing Your Data

The plugin accepts data in Excel (.xlsx, .xls) or CSV format. Here's how to structure your data:

  1. Headers row: The first row of your spreadsheet should contain column headers that will be mapped to layer names in your Figma design.
  2. Data rows: Each subsequent row represents a single banner variant.
  3. Data types:
    • Text: Any text content that will be applied to text layers
    • Images: Direct HTTP/HTTPS URLs to images that will be applied to layers with image fills. We recommend using https://postimages.org/ for hosting your images. Make sure to copy the "Direct Link" URL when uploading.
    • Visibility: Boolean values ("true"/"false", "yes"/"no", "1"/"0") to control layer visibility
    • Colors: Hex codes (#RRGGBB) or RGB/RGBA values for fill colors
  4. Special considerations:
    • Price formatting: For detailed price control, you can have separate columns for main price, decimal, and currency
    • Boolean values: The plugin intelligently detects boolean-like values and can use them to control layer visibility
    • URLs: The plugin processes image URLs, optimizing and resizing them as needed

Using the Plugin

Step 1: Upload Data

Launch the plugin from the Plugins menu in Figma. In the initial screen, you have three options to upload your data:

  • Drag and drop an Excel/CSV file into the upload area
  • Click "Browse Files" to select a file from your computer
  • Paste data directly from your clipboard (e.g., from Excel or Google Sheets)

Step 2: Configure and Generate

After uploading your data, the plugin will display the configuration screen:

Select Template:

  • Browse through available pages in your Figma document
  • Click on a page to select it as your template
  • The plugin will scan the page for compatible layers

Map Data to Layers:

  • The plugin automatically suggests mappings based on layer and column names, and is almost always successful with auto-mapping
  • Custom mapping is primarily needed for debugging errors or special cases
  • To customize mappings if needed, check "Customize layer mapping"
  • For each layer, select the appropriate data column from the dropdown

Generate Banners:

  • Once you're satisfied with the mapping, click "Generate Banners"
  • The plugin will create duplicates of your template for each row in your data
  • It will apply the mapped data to the appropriate layers in each duplicate
  • The generated banners will appear on a new page called "✅ Generated variants"

Export Generated Banners:

  • After generation completes, the export options will appear
  • Select the desired format (PNG, JPG, SVG, PDF) and scale
  • Choose which frames to export using the checkboxes
  • Click "Download Selected Frames (ZIP)" to export

Export Options

The plugin provides flexible export options:

  • Format: Choose between PNG, JPG, SVG, or PDF (PNG or JPG at 1x scale is recommended for most uses)
  • Scale: Select from 1x, 2x, 3x, or 4x resolution (1x is recommended for standard use)
  • Frame selection: Choose which specific frames to export
  • Output: Exports are packaged in a ZIP file with an organized folder structure:
    • Root folder named [project_name]_export_[MM-DD_HH MM] (using current timestamp)
    • Each generation session gets a subfolder with its timestamp
    • Inside each session folder, rows are organized in row_[XX] folders
    • Individual banner files are named based on the variant with clean file names, preserving capitalization
    • File extensions match the selected format (.png, .jpg, etc.)

Advanced Features

Intelligent Data Type Detection

The plugin automatically detects the appropriate mapping type based on your data:

  • URLs are mapped to image layers
  • Boolean values (true/false, yes/no, 1/0) are mapped to layer visibility
  • Text content is mapped to text layers
  • Color codes are mapped to fills

Automated Price Formatting

For price display, the plugin offers advanced formatting:

  • Split price values into main number, decimal, and currency components
  • Auto-adjusts font size for prices with 3+ digits
  • Maintains consistent scaling across all price components

Smart Template Selection

The plugin intelligently skips generating certain templates based on data:

  • Skips templates with "Discount" layers when discount data is empty
  • Skips templates with "New_price" layers when new price data is empty
  • Handles various price display scenarios based on available data

Price and Discount Rules

The plugin follows specific rules when deciding which templates to generate based on price and discount data:

  • Rule 1: If a template contains a "Discount" layer and the discount data field is empty, that template will be skipped
  • Rule 2: If a template contains a "New_price" layer but the new price data is empty, that template will be skipped
  • Rule 3: If a template has both "New_price" and "Regular_price" layers, but only new price data is available (regular price is empty), that template will be skipped
  • Rule 4: If a template has only a "New_price" layer (no "Regular_price" layer), but both new price and regular price data are available, that template will be skipped

Tip

These rules allow you to create different template designs for various pricing scenarios (regular price only, discount price only, both prices together) and the plugin will automatically select the appropriate template based on the available data for each row.

Image Processing

The plugin optimizes images before applying them:

  • Automatically checks image dimensions
  • Automatically resizes images over 4000px to comply with Figma's limits
  • Caches processed images to improve performance
  • Handles various image formats and transparency options

Intelligent Text Sizing

The plugin automatically adjusts text size based on content length:

  • For Offer_name layers, scales up text size for short text (25 or fewer characters get 1.5x size increase, 26-35 characters get 1.2x)
  • For price layers, reduces font size by 5% when the main price has 3 or more digits
  • Maintains consistent scaling across related components (main number, decimal, currency)

Best Practices

  1. Consistent naming: Use consistent naming conventions between your spreadsheet columns and Figma layers to enable automatic mapping.
  2. Layer organization: Keep your template designs clean and well-organized, with properly named layers.
  3. Test with sample data: Before processing large datasets, test with a few rows to ensure your mappings work as expected.
  4. Image dimensions: For best results, prepare images with appropriate dimensions before including their URLs in your data.
  5. Boolean naming: For layers that should be toggled on/off, use clear yes/no or true/false values in your data.
  6. Template variations: Create multiple template designs to handle different data scenarios (with/without discounts, with/without partner logos, etc.).

Troubleshooting

Common Issues

Plugin doesn't recognize layers:

  • Ensure layers are properly named and match column headers
  • Check that layers are not locked or hidden
  • Verify that text layers can be edited

Images not displaying correctly:

  • Verify image URLs are valid and accessible
  • Check that the layer can accept image fills
  • Ensure the layer is not locked or has effects that might interfere with fills
Tip

If images do not generate, check the image links in your data file. They MUST be direct links. A direct link format looks like this: https://i.postimg.cc/W3KhFzBT/TVBOKSTAS-3.jpg (notice the image filename is included in the URL). When using postimages.org, be sure to copy the "Direct link" URL, not the page URL.

Generation taking too long:

  • For large datasets, try generating in smaller batches
  • Optimize image URLs to point to appropriately sized images
  • Close other resource-intensive applications

Export issues:

  • Ensure you have selected at least one frame to export
  • Try a different export format if one is not working
  • For large exports, be patient as the ZIP file is created

Getting Help

This step-by-step tutorial will guide you through the process of using the GD Banner Automator Figma plugin to create multiple banner variants from a single template and data source.

Prerequisites

Before starting this tutorial, make sure you have:

  1. The GD Banner Automator plugin installed in Figma (see Installation section and download from SharePoint)
  2. A Figma document with banner templates
  3. Data in Excel (.xlsx) or CSV format, or data copied to your clipboard

Tutorial Overview

We'll walk through the complete workflow:

  1. Preparing your Figma template
  2. Structuring your data spreadsheet
  3. Using the plugin to generate banner variants
  4. Exporting the generated banners

Let's get started!

Step 1: Prepare Your Figma Template

A well-prepared template is crucial for successful automation. Let's set up a template page:

  1. Create a dedicated page for your template designs:
    • Click the "+" icon next to Pages in the Figma sidebar
    • Name the page something descriptive like "Banner Templates"
  2. Design your banner template(s) within this page:
    • Create frames for each banner variation you need
    • Add all necessary design elements (text, shapes, images)
  3. Name your layers correctly to match your data:
    • For text layers that will change, use names that match your data columns
    • For image layers, use names that match the column with image URLs
    • For visibility-controlled elements, use names that match boolean columns
  4. Set up special price layers if needed:
    • For main price: price_Price_main
    • For decimal part: price_Price_decimal
    • For currency symbol: price_Price_currency
  5. Test your template design to ensure all elements are positioned correctly and all text is editable.
Tip

Make sure your layer names are descriptive and easy to match with your data columns. Clear naming makes the mapping process much smoother!

Step 2: Structure Your Data Spreadsheet

Now, let's prepare your data file. You can use Excel, Google Sheets (export as xlsx), or any spreadsheet software that can produce Excel or CSV files.

  1. Create a spreadsheet with the following structure:
    • First row: Column headers that match your layer names
    • Subsequent rows: Data for each banner variant
  2. Example spreadsheet columns:
    • Product_Name - Text content for product name layer
    • Offer_Description - Text content for offer description layer
    • Product_Image - Direct URLs to product images (use postimages.org and copy the "Direct link" URL)
    • Price - Price values (e.g., "12.99€")
    • Discount - Discount values or boolean for showing discount
    • Show_Logo - Boolean values ("true"/"false") to control logo visibility
  3. Formatting considerations:
    • Text: Regular text entries (keep length appropriate for your design)
    • Images: Full HTTP/HTTPS URLs to accessible images
    • Visibility: Boolean values ("true"/"false", "yes"/"no", or "1"/"0")
    • Colors: Hex codes (#RRGGBB) or RGB values (rgb(255,0,0))
  4. Save your spreadsheet as .xlsx, .xls, or .csv format.
Example spreadsheet content: 
Product_Name,Offer_Description,Product_Image,Price,Discount,Show_Logo

"Sunglasses","Bestseller","https://example.com/sunglasses.jpg","39.99€","15%","true"

Step 3: Launch the Plugin and Upload Data

Now, let's use the plugin to generate banner variants:

  1. Open your Figma document with the template page.
  2. Launch the GD Banner Automator plugin:
    • Click on the menu icon (≡) in the top-left corner
    • Navigate to Plugins > Development > GD Banner Automator
    • (If you haven't installed the plugin yet, download it from SharePoint and follow the installation instructions)
  3. Upload your data using one of these methods:
    • Drag and drop your Excel/CSV file into the upload area
    • Click "Browse Files" to select your file
    • Copy data from your spreadsheet and click anywhere in the upload area to paste
  4. Wait for the data to process - the plugin will read your file and prepare it for mapping.

Step 4: Map Data to Layers and Generate

After uploading your data, you'll see the configuration screen:

  1. Select template:
    • Browse through the available pages in the grid
    • Click on your template page to select it
    • The plugin will scan the page for compatible layers
  2. Review automatic mappings:
    • The plugin tries to match column names to layer names
    • For custom mapping, check "Customize layer mapping"
    • For each layer, select the appropriate data column from the dropdown
  3. Generate banners:
    • Once mapping is set up, click "Generate Banners"
    • The plugin will show a loading indicator while processing
    • Wait for the generation to complete
  4. Review generated banners:
    • The plugin will create or use a page called "✅ Generated variants"
    • All banner variants will be organized on this page
    • Each row from your data will have corresponding banner variants
Important

If the plugin skips generating certain templates, check the generation summary. It will show skip reasons (e.g., missing discount data for a template with discount layers).

Step 5: Export the Generated Banners

After generating your banners, you can export them as image files:

  1. Manually review and adjust banners if needed:
    • Before exporting, take time to review the generated variants
    • Make manual corrections for photo positioning, cropping, or text breaking issues
    • Adjust layer properties if needed (size, position, opacity, etc.)
    • Check that text flows properly and doesn't overflow its containers
  2. Configure export options:
    • Select the desired format (PNG or JPG at 1x scale is recommended for most uses)
    • Choose the scale (1x is recommended for standard use)
    • Select which frames to export using the checkboxes
  3. Initiate export:
    • Click "Download Selected Frames (ZIP)"
    • Wait for the export process to complete
    • The plugin will show progress updates during export
  4. Save the ZIP file:
    • When export completes, your browser will download a ZIP file
    • The file contains all exported banners organized in folders
    • Folders are structured by generation timestamp and row number
Tip

While the plugin does its best to automate banner creation, a quick manual review before export can help catch any issues that might need fine-tuning, especially with complex layouts or variable-length content.

Additional Tips and Tricks

Handling Different Banner Types

If you have different banner formats (e.g., square, horizontal, vertical):

  1. Place all template variations on the same template page
  2. The plugin will generate variants for each template using the same data
  3. Templates without matching data will be skipped automatically

Working with Price Formatting

For better control over price display:

  1. Split your price data into components in your spreadsheet:
    • Price_Main (e.g., "12")
    • Price_Decimal (e.g., "99")
    • Price_Currency (e.g., "€")
  2. Name your layers accordingly:
    • price_Price_Main_main
    • price_Price_Decimal_decimal
    • price_Price_Currency_currency

Optimizing Image Usage

For better performance with images:

  1. Use postimages.org for hosting your images
  2. Always copy the "Direct link" URL after uploading
  3. Use web-optimized formats (JPEG, PNG) with reasonable file sizes
  4. Images over 4000px will be automatically resized by the plugin, but for best performance, prepare images at appropriate dimensions before uploading

Updating Generated Banners

If you need to make changes:

  1. Modify your data and/or template as needed
  2. Run the plugin again to generate a new set of banners
  3. Each generation creates a new container on the Generated variants page

Troubleshooting During the Process

Issue: Layers not updating correctly

Solution:

  • Check if layer names exactly match column headers (case-sensitive)
  • Ensure layers are not locked or hidden
  • Verify text layers have editable text content

Issue: Images not displaying

Solution:

  • Check if image URLs are valid and accessible
  • Ensure image layers can accept fills (rectangles, frames, etc.)
  • Try different image formats if one isn't working

Issue: Missing variants

Solution:

  • Check if templates with "Discount" layers are being skipped due to empty discount data
  • Verify that price data matches the expected format for price-related templates
  • Review the generation summary for skip reasons
Conclusion

Congratulations! You've learned how to use the GD Banner Automator plugin to automate banner creation in Figma. This workflow can save you hours of repetitive work when creating multiple banner variants.

Remember that you can always return to this tutorial as a reference when using the plugin for future projects.
